Fachliche Dokumentation

Formal kann der DEA als 5-Tupel definiert werden:
E = (Q, Σ, δ, q0, F)

Die Zustandsmenge Q lautet {1, 2, 3, 4, 5, 6, 7, 8}.
Das Eingabealphabet Σ umfasst {B, E, P, S, T, V, X}.
Der Startzustand q0 ist {1} und die Menge an Endzuständen F enthält {8}. 
Die Übergangsfunktion δ kann als Tabelle dargestellt werden:

Technische Dokumentation

Das User Interface habe ich mithilfe der Entwicklungsanwendung Bootstrap Studio gestaltet. 

Für die Visualisierung des Automaten habe ich das XState Visualizer Tool verwendet.
Diesen habe ich als Inlineframe in meinen HTML-Code der Startseite eingebunden. 

Für das Erstellen der zufälligen Zeichenketten habe ich zwei kleine JavaScript-Anwendungen geschrieben.
Hierfür habe ich ein Array mit 25 korrekten und ein Array mit 25 falschen Zeichenketten erstellt. Aus diesen wird ein zufälliges Beispiel ausgewählt und ausgegeben. Beim erneuten Klicken der Buttons erscheint jeweils ein neues Beispiel.
